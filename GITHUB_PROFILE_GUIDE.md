## 3. Create Your Profile README

GitHub has a special hidden feature: if you create a repository named **exactly the same as your username**, the `README.md` inside it appears at the top of your profile — like a cover letter pinned above everything else.

### How to set it up

1. Go to [https://github.com/new](https://github.com/new)
2. Set **Repository name** to your exact GitHub username (e.g., if your username is `saifkhan`, the repo name must also be `saifkhan`)
3. GitHub will show a green sparkle banner: *"✨ You found a secret!"* — that confirms you did it right
4. Set visibility to **Public**
5. Check **"Add a README file"**
6. Click **Create repository**
7. Click the pencil ✏️ icon to edit the `README.md`
8. Delete all existing content and paste the template from Section 4

## 5. Pin Your Best Repositories

Pinned repositories appear at the very top of your profile — they are the **first thing every client sees** before anything else on your page.

### How to pin repositories

1. Go to your GitHub profile page (`github.com/YOUR_USERNAME`)
2. In the **Pinned** section, click **Customize your pins**
3. Select up to **6 repositories** (private repos can also be pinned)
4. Drag to reorder — put your strongest work first
5. Click **Save pins**

### Which repositories to pin and in what order

| Position | Repository | Why |
|---|---|---|
| **1st** | `devstudio` | Your flagship — full-stack, AI-powered, production deployed |
| **2nd** | A custom web app for a real client | Proves client delivery |
| **3rd** | A project using PostgreSQL + Node.js backend | Shows backend depth |
| **4th** | An AI/LLM integration project | Your fastest-growing service |
| **5th** | A Next.js or SaaS project | Shows modern framework experience |
| **6th** | An open-source utility or package | Signals community contribution |

> Quality over quantity. 3 polished repos with live demos beat 10 half-finished projects every time.

## 8. Grow Your Contribution Graph

The green contribution graph shows how consistently you write code. A consistent graph builds trust — it tells clients you are reliable, active, and professional.

### The green square rule

Every green square = at least one Git commit pushed that day. You do not need to work on a big project to stay active.

### Daily habits to stay active

| Action | Takes | Counts as |
|---|---|---|
| Fix a bug in any repo | 5–15 mins | Contribution |
| Update a README or docs | 2–5 mins | Contribution |
| Add comments or refactor code | 10 mins | Contribution |
| Push a feature to a personal project | 30–60 mins | Contribution |
| Commit to a private client repo | Any time | Contribution (if private contributions are enabled) |

### Project ideas to keep your graph active between client work

- **DevStudio improvements** — UI tweaks, new portfolio projects, performance improvements
- **Open source contributions** — fix a bug or add docs to a popular npm package
- **Personal tools** — build small CLI tools, utilities, or scripts you actually use
- **Learning projects** — new AI API, new database, new framework — commit as you go

> Work in small, frequent commits rather than one massive push per week. This fills your graph and shows good Git habits.
