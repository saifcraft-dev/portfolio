Phase 1: Project Foundation (Steps 1-5)
Step 1: Project Initialization
Create a new Next.js 14+ project with App Router and TypeScript. Use the command: npx create-next-app@latest dev-team-portfolio --typescript --tailwind --app --eslint --src-dir. Set up the basic folder structure: src/app, src/components, src/lib, src/types, and src/context. Initialize git repository and create initial commit.

Step 2: Core Dependencies Installation
Install essential packages: npm install firebase @radix-ui/react-icons class-variance-authority clsx tailwind-merge lucide-react framer-motion react-hook-form @hookform/resolvers zod. These provide Firebase SDK, UI icons, utility functions, animations, and form handling capabilities.

Step 3: Shadcn/UI Setup
Initialize Shadcn/UI with npx shadcn-ui@latest init. Configure with default settings and install core components: npx shadcn-ui@latest add button card input textarea label select dialog sheet table badge avatar toast form dropdown-menu tabs alert. These components will provide the UI foundation.

Step 4: Environment Configuration
Create .env.local file with Firebase configuration variables (all prefixed with NEXT_PUBLIC_ for client access): NEXT_PUBLIC_FIREBASE_API_KEY, NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN, NEXT_PUBLIC_FIREBASE_PROJECT_ID, NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET, NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID, NEXT_PUBLIC_FIREBASE_APP_ID, NEXT_PUBLIC_CLOUDINARY_CLOUD_NAME, CLOUDINARY_API_KEY, CLOUDINARY_API_SECRET. Add .env.local to .gitignore.

Step 5: TypeScript Definitions
Create src/types/index.ts with interfaces for: Project (id, title, description, longDescription, technologies, imageUrl, projectUrl, githubUrl, category, featured, completedDate, createdAt, updatedAt), Service (id, title, description, features, pricing, deliveryTime, category, active), Order (id, clientName, clientEmail, serviceType, projectDescription, budget, timeline, status, priority, createdAt, updatedAt, notes), TeamMember (id, name, role, bio, imageUrl, skills, socialLinks, active), and enums for OrderStatus, OrderPriority, ProjectCategory.

Phase 2: Firebase Integration (Steps 6-10)
Step 6: Firebase Configuration
Create src/lib/firebase/config.ts to initialize Firebase app, auth, and firestore instances using environment variables. Export configured instances for use throughout the application. Implement proper error handling for initialization failures.

Step 7: Authentication Setup
Create src/lib/firebase/auth.ts with functions: signInWithEmail(email, password), signInWithGoogle(), signOut(), getCurrentUser(), and isAdmin(user). Implement admin checking by comparing user email against a whitelist stored in environment variables or Firestore.

Step 8: Firestore Database Schema
Design Firestore collections: projects, services, orders, team. Create src/lib/firebase/firestore.ts with CRUD functions for each collection. Implement proper error handling and type safety for all database operations.

Step 9: Firestore Security Rules
Configure Firestore security rules to allow public read access to projects, services, and team collections. Restrict orders collection to allow public create access but admin-only read/update access. Implement admin verification in security rules using custom claims or email whitelist.

Step 10: Authentication Context
Create src/context/AuthContext.tsx providing authentication state management with useAuth hook. Include user state, loading state, admin status, and authentication functions. Wrap the app in this context provider.
