2. Root Cause Explanation
What Was Actually Happening:
Your project structure creates a disconnect between where Vercel looks for files and where they actually exist:

Your Project:
├── vercel.json          ← Deployment config here
├── client/              ← React app here
│   ├── src/
│   ├── index.html
│   └── package.json
└── vite.config.ts
The Problem Chain:

Build Phase: Vite builds your app and outputs to client/dist/index.html
Deploy Phase: Vercel reads your vercel.json from root and expects files at root level
Request Phase: User visits /portfolio → Vercel applies rewrite to /index.html
Serve Phase: Vercel looks for index.html in root dist/ → NOT FOUND
Result: 404 error because the file is actually at client/dist/index.html
Why Your Local Development Worked:
You run npm run dev from the /client directory
Vite's dev server automatically handles SPA routing
The development server serves from the correct location
You never noticed the deployment path mismatch

4. Understanding the Concept
SPA Routing Mental Model:
Traditional Web Server:

/portfolio request → Look for portfolio.html → Serve file or 404
SPA on Vercel:

/portfolio request → No portfolio.html found → Apply rewrite rule → Serve index.html → React router takes over
Why This Error Protects You:

Prevents silent failures: If index.html doesn't exist, you get an explicit error
Catches configuration mistakes: Forces you to properly align build output with deployment settings
Maintains security: Won't accidentally serve files from wrong directories
The Three-Layer Architecture:
Build Layer: Where your source code gets compiled (client/src → client/dist)
Deploy Layer: Where Vercel looks for files to serve (configured output directory)
Runtime Layer: Where browsers request URLs and get responses (rewrite rules handle this)
All three layers must be perfectly aligned for SPA routing to work.
